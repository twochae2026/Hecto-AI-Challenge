# Hecto-AI-Challenge
2025 하반기 헥토 채용 AI 경진대회 260127~260202(10:00am)
## 2. 대회 규칙

### 2.1 이미지 단위 입력 처리 요건
- 영상(Video) 데이터는 프레임 단위로 분해하여 각 프레임을 개별 이미지로 모델에 입력해야 합니다.
- 복수 프레임을 동시에 입력하여 시간적 정보를 직접 활용하는 방식은 허용되지 않습니다.
- 단, 프레임별 독립 추론 결과를 종합하여 최종 결과를 산출하는 후처리 과정은 허용됩니다.

### 2.2 사전 학습 모델 사용 가능 범위
- 2025년 12월 29일 전(~2025.12.28)에 공식적으로 가중치가 공개된 사전 학습 모델만 사용 가능.
- 해당 모델은 최소 비상업적 이용(NC)이 허용된 라이선스를 따라야 함.
- 모델 사용 가능 여부 판단 기준:
  - 아키텍처 라이선스가 비상업적 이용조차 허용하지 않으면 사용 불가.
  - 사전 학습 과정에서 사용된 데이터셋 라이선스와는 무관.

### 2.3 외부 데이터 사용 가능
- 공식 학습 데이터 미제공으로, 참가자가 직접 데이터 수집/생성 가능.
- 모든 데이터는 저작권·개인정보보호 등 관련 법령 준수 필수.
- API 활용 가능하나, 모델 학습·추론에 직접 사용 불가.
- 외부 데이터 사용 시:
  - 데이터 출처 및 생성 방식 명시
  - 법적 문제 없음을 증빙할 수 있어야 함

### 2.4 API 사용 제한
- 원격 서버 기반 API(예: OpenAI API, Gemini API, 딥페이크 판별 서비스 API 등)는 모델 학습/추론 과정에 직접 사용 불가.
- 단, 학습 데이터 수집/생성을 위한 API 사용은 허용:
  - API 호출은 학습·추론 단계에서 직접 사용 불가
  - 생성된 데이터는 정적 형태로 저장 후 활용
  - 외부 데이터 사용 규칙 준수 필수

### 2.5 단일 모델 구성 요건
- 추론은 반드시 단일 모델 사용
- 복수 모델/백본 결합, 독립 학습 모델의 결과 결합, TTA 등 반복 추론 후 결과 결합 방식 불허

#### 2.5.1 허용 범위
- 고정 모델 아키텍처 기반 단일 사전학습 모델 추론
- 단일 백본으로부터 feature 추출 후 head 확장 또는 auxiliary head 구성

#### 2.5.2 불허 사항
- 복수 모델 또는 백본 병렬 결합
- 독립 학습 모델 예측 결과/임베딩 결합
- 입력 특성에 따라 모델/백본 선택·전환 구조
- Test-Time Augmentation(TTA) 등 반복 추론 후 결과 결합

### 2.6 학습·추론 코드 구성 요건

#### 2.6.1 학습 코드
- 모든 GPU에서 48GB VRAM 내 정상 작동

#### 2.6.2 추론 코드
- 운영진 환경 내 제한 시간 내 정상 작동
- 전체 평가 데이터 기준 총 추론 시간: 1시간 초과 불가
  - 동일 환경에서 5회 실행 평균 소요 시간으로 산정

#### 2.6.3 추론 환경(리소스)
- GPU: L40S (48GB VRAM)
- CPU: 16 vCPUs (96GB RAM)
- DISK: 80GB
- CUDA: 11.8 ~ 12.6
- PyTorch: 2.5.0 권장

#### 2.6.4 추론 코드 유의 사항
- 코드에 반드시 포함: `데이터 전처리` → `모델 로드` → `모델 추론` → `최종 결과 생성`

#### 2.6.5 실행 환경 제약
- 오프라인 환경, 인터넷 연결 차단
- 외부 서버/인터넷 접근 불가
